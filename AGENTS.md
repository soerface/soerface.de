# Agent Guidelines for soerface.de

This file contains knowledge and guidelines for AI coding agents working on this project.

## Tech Stack
- **Web Framework:** Flask
- **Templating:** Jinja2
- **Static Site Generator:** Frozen-Flask (`freeze.py`)
- **Package Manager:** `uv`
- **Frontend:** Bootstrap 4, jQuery, Custom CSS

## Project Structure
- `sections/`: Contains Flask Blueprints for different parts of the site (e.g., `talks.py`, `tools.py`, `blog`).
- `templates/`: Jinja2 templates. `base.html` is the primary layout. Sub-directories match section names.
- `data/`: Contains content/data for the dynamic sections.
- `static/`: Site-wide static assets (CSS, JS, fonts).
- `env.py`: Defines core environment variables and paths like `DATA_PATH`.
- `build/`: The output directory generated by Frozen-Flask.

## Key Concepts

### Frozen-Flask Compatibility
Because the site is compiled to a static site using Frozen-Flask, you must ensure that all dynamically served files have corresponding `url_for` calls during the index route generation, otherwise `freeze.py` will not discover and build them.
Example from `sections/tools.py`:
```python
for path in (env.DATA_PATH / "tools" / tool["path"]).rglob("*"):
    if path.is_file() and path.name != "index.html":
        rel_path = path.relative_to(env.DATA_PATH / "tools" / tool["path"])
        url_for(f"tools.{tool['path']}_static", filename=rel_path)
```

## Common Commands
- **Build the static site:** `uv run freeze.py`
